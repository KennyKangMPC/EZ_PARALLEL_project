subroutine decomp_grid_dble ( integer, intent(in)  row_count,
integer, intent(inout)  col_count,
integer, intent(in)  overlap,
integer, intent(in)  decomp_id,
double precision, dimension(:,:), intent(in), allocatable  grid 
)

The EZ_PARALLEL grid decomposition subroutine for DOUBLE PRECISION grids. Given the column count of the grid, returns the column count of the sub-grid, including overlap (for use of the vertical slab decomposition). Also stores the data in the grid_decomps list for use in other subroutines.

Author
Jason Turner, University of Wisconsin-Madison
Parameters
[in]row_countNumber of rows in the grid.
[in,out]col_countNumber of columns in the grid, changes to number of columns in the sub-grid including overlap, so that the user's time-stepping code may remain unchanged.
[in]overlapNumber of extra columns needed by each sub-grid to successfully step forward in time.
[in]decomp_idID number of the grid decomposition, between 1 and the number of "unique" grid decompositions.
[in]gridThe grid to be allocated and decomposed, for use in identifying what datatype it is. NOTE: It must not yet be allocated.

subroutine decomp_grid_dble::error_handling_dble ( integer, intent(in)  row_count,
integer, intent(in)  col_count,
integer, intent(in)  overlap,
integer, intent(in)  decomp_id 
)

The EZ_PARALLEL grid decomposition error handling subroutine. Aborts the program if the grid or grid decomposition parameters trigger any of the following errors:

  • row_count...
    1. is not positive
  • col_count...
    1. is not positive
    2. is too small to be divided among the processors, ignoring the overlap
    3. is too small to be divided among the processors, including overlap
  • overlap...
    1. is negative
  • decomp_id...
    1. is outside of (1, ..., decomp_count)
Parameters
[in]row_countNumber of rows in the grid.
[in]col_countNumber of columns in the grid, changes to number of columns in the sub-grid including overlap, so that the user's time-stepping code may remain unchanged.
[in]overlapNumber of extra columns needed by each sub-grid to successfully step forward in time.
[in]decomp_idID number of the grid decomposition, between 1 and the number of "unique" grid decompositions.
