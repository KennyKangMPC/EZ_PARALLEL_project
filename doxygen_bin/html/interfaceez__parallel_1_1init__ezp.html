Detailed Description

The initialization interface for the EZ_PARALLEL module.

The INIT_EZP interface initializes the EZ_PARALLEL module by intializing MPI, allocating memory to the list of grid decompositions, and storing variables frequently used throughout the module subroutines. For greater detail on the initialization subroutine, please see init_ezp.f90. For greater detail on the variables frequently used throughout the module subroutines (which are accessible for read/write by the user), please see ez_parallel_structs.f90.

Remarks
Two grid decompositions differ for the sake of decomp_count_L if they differ in any of:
  • size (the number of rows and/or columns),
  • overlap parameter (the number of columns one sub-grid must borrow from another to perform a time-step),
  • datatype they contain (DOUBLE PRECISION or DOUBLE COMPLEX).
Hence, if the user will need three grid decompositions, they should use CALL INIT_EZP(3).
