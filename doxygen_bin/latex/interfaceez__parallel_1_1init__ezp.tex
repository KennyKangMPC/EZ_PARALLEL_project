\doxysubsection{Detailed Description}
The initialization interface for the {\ttfamily E\+Z\+\_\+\+P\+A\+R\+A\+L\+L\+EL} module. 

The {\ttfamily I\+N\+I\+T\+\_\+\+E\+ZP} interface initializes the {\ttfamily E\+Z\+\_\+\+P\+A\+R\+A\+L\+L\+EL} module by intializing {\ttfamily M\+PI}, allocating memory to the list of grid decompositions, and storing variables frequently used throughout the module subroutines. For greater detail on the initialization subroutine, please see \mbox{\hyperlink{init__ezp_8f90}{init\+\_\+ezp.\+f90}}. For greater detail on the variables frequently used throughout the module subroutines (which are accessible for read/write by the user), please see \mbox{\hyperlink{ez__parallel__structs_8f90}{ez\+\_\+parallel\+\_\+structs.\+f90}}. \begin{DoxyRemark}{Remarks}
Two grid decompositions differ for the sake of {\ttfamily decomp\+\_\+count\+\_\+L} if they differ in any of\+: \begin{DoxyItemize}
\item size (the number of rows and/or columns), \item overlap parameter (the number of columns one sub-\/grid must borrow from another to perform a time-\/step), \item datatype they contain ({\ttfamily D\+O\+U\+B\+LE P\+R\+E\+C\+I\+S\+I\+ON} or {\ttfamily D\+O\+U\+B\+LE C\+O\+M\+P\+L\+EX}).\end{DoxyItemize}
Hence, if the user will need three grid decompositions, they should use {\ttfamily C\+A\+LL I\+N\+I\+T\+\_\+\+E\+Z\+P(3)}. 
\end{DoxyRemark}
